# Model Card — Qwen2.5-0.5B Structured Extraction

## Model Description

A QLoRA fine-tuned adapter on top of **Qwen2.5-0.5B-Instruct** that converts unstructured business text into normalized, schema-compliant JSON. The adapter is trained with two stages: **SFT** (supervised fine-tuning) followed by **DPO** (direct preference optimisation) to sharpen output quality and eliminate hallucinations.

| Attribute | Value |
|-----------|-------|
| Base model | `Qwen/Qwen2.5-0.5B-Instruct` |
| Adapter type | LoRA (rank 8, alpha 16, RSLoRA) |
| Training stages | QLoRA SFT → DPO |
| Training hardware | NVIDIA GTX 1650 (4 GB VRAM) |
| Quantization | 4-bit NF4 (double quant) |
| Languages | English |
| License | Apache 2.0 |

## Intended Use

**Primary use:** Extract structured JSON from English business text (press releases, earnings summaries, deal announcements). Output fields: `event_type`, `entities`, `dates`, `financials`, `relationships`, `metrics`.

**Intended users:** Developers building data pipelines that consume unstructured news or document text and need normalized entity/event data.

**Out-of-scope uses:** Non-English text, long documents (>8 000 characters), medical/legal/financial advice, generation of original content.

## Evaluation Results

Evaluated on 30 held-out examples (base model vs. SFT vs. DPO):

| Metric | Base | SFT | DPO |
|--------|------|-----|-----|
| Entity F1 | 0.737 | 0.990 | **1.000** |
| Entity Recall | 0.674 | 0.982 | **1.000** |
| Entity Precision | 0.892 | 1.000 | **1.000** |
| Date Normalization Recall | 0% | **100%** | **100%** |
| Financial Recall | 12.5% | **100%** | 94.1% |
| Avg Hallucinations / output | 0.75 | **0.0** | **0.0** |
| Valid JSON Rate | 80% | 37% | 57% |

> **On Valid JSON Rate:** The base model produces syntactically valid JSON but with zero schema compliance. SFT/DPO outputs are occasionally truncated at `max_new_tokens=512`; increasing to 768+ closes this gap.

## Training Data

5 000 synthetic SFT examples and 2 000 DPO preference pairs generated by `data/generate_dataset.py` across 7 business event scenarios:

- Funding rounds (VC/PE investment announcements)
- M&A acquisitions
- Executive hires and departures
- Product launches
- Strategic partnerships
- Quarterly earnings results
- Legal/regulatory events

All data is synthetic (template-based with randomised entities). No real personal data or proprietary information is included.

## Limitations

- **Token budget:** Full extraction JSON requires ~370 tokens; outputs may truncate on complex inputs with `max_new_tokens=512`. Increase to 768 for production use.
- **Synthetic training data:** The model has been trained on template-generated text. Real-world text with unusual phrasing may produce lower-quality extractions.
- **English only:** Not tested on multilingual input.
- **4-bit quantization:** Inference uses NF4 quantization for VRAM efficiency; full-precision inference would improve accuracy marginally.

## Bias and Fairness

The training data uses a fixed pool of synthetic person names, organizations, and locations. The model has not been evaluated for demographic bias in entity extraction. Users should audit outputs for their specific domain before production deployment.

## How to Use

```python
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import PeftModel

base = AutoModelForCausalLM.from_pretrained("Qwen/Qwen2.5-0.5B-Instruct")
model = PeftModel.from_pretrained(base, "./outputs/dpo/adapter")
tokenizer = AutoTokenizer.from_pretrained("./outputs/dpo/adapter")

text = "Acme Corp raised $50M Series B led by Example Ventures on March 3, 2026."
messages = [{"role": "user", "content": f"Extract structured data as JSON:\n\n{text}"}]
prompt = tokenizer.apply_chat_template(messages, tokenize=False, add_generation_prompt=True)
inputs = tokenizer(prompt, return_tensors="pt")
output = model.generate(**inputs, max_new_tokens=512, do_sample=False)
print(tokenizer.decode(output[0][inputs["input_ids"].shape[1]:], skip_special_tokens=True))
```
